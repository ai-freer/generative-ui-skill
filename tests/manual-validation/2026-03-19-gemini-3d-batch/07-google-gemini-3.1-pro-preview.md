### 补充说明
* **无状态的优势**：因为服务端不需要维护 Session 记录，后端可以轻松进行水平扩展（多台服务器之间不需要同步登录状态），只要它们拥有相同的密钥（Secret）即可验证签名。
* **数据自包含**：JWT 的 Payload 中可以携带非敏感的用户信息（如 User ID、角色权限等），使得服务端能直接从 Token 中解析出上下文，减少了查询数据库的次数。
* **安全性建议**：JWT 一旦签发，在过期前默认是有效的（难以主动吊销）。因此通常将 Access Token 的过期时间设置得很短（如 15 分钟），并配合有效期较长的 Refresh Token 来兼顾安全与用户体验。
